## üõ†Ô∏è Getting Started

### 1. Fork the Repository

Click **Fork** on GitHub to create your own copy of the project.

### 2. Clone Your Fork

```
git clone https://github.com/<your-username>/olang.git
cd olang
```

### 3. Create a New Branch

```
git checkout -b feature/my-new-feature
```

### 4. Make Your Changes

Write clean, readable code and include tests if possible.

### 5. Commit Your Changes

```
git commit -m "Add: description of the feature or fix"
```

### 6. Push to Your Fork

```
git push origin feature/my-new-feature
```

### 7. Open a Pull Request

Go to your fork on GitHub and click **Pull Request**.

---

## ‚úî Pull Request Guidelines

* Keep PRs focused on one change at a time
* Write clear commit messages
* Include documentation updates if necessary
* Ensure code passes all tests
* Be open to feedback ‚Äî maintainers may request updates

Your PR will be reviewed by the project maintainer or trusted reviewers.

## üêõ Reporting Issues

If you find a bug:

1. Go to **Issues**
2. Choose "Bug Report"
3. Include:

   * Steps to reproduce
   * Expected behavior
   * What actually happened
   * Error logs or screenshots if available

## üí° Proposing New Features

If you want to suggest something new:

1. Open a **Feature Request**
2. Explain the problem
3. Describe your proposed solution
4. Add examples or pseudo-code if possible

Major features are discussed publicly before implementation.
